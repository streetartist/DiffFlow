# 🚀 DiffFlow 实时协作插件：完全使用手册

DiffFlow 是一款为 Godot 4 定制的实时协同插件。它通过服务端账号、项目权限和文件同步通道，让团队成员在同一个项目中保持文件状态一致。

---

## 一、 环境准备与安装

### 1. 插件安装
1. 将项目中的 `addons/diffflow` 文件夹完整拷贝到您的 Godot 项目根目录下。
2. 打开 Godot 编辑器，进入 **项目 (Project) -> 项目设置 (Project Settings) -> 插件 (Plugins)**。
3. 找到 `DiffFlow`，勾选 **启用 (Enable)**。

### 2. 登录并选择项目
1. 启用插件后，编辑器右侧面板会出现 **DiffFlow** 选项卡。
2. **服务器地址**：输入服务端 HTTP 地址（例如 `http://您的服务器IP:8090`）。
3. 输入管理员分配的账号和密码，点击 **登录**。
4. 如果管理员给的是授权密钥，填写账号、密码和授权密钥后点击 **注册**。
5. 从项目下拉框中选择要参与的项目。
6. 点击 **进入项目**。插件会先执行首次同步，完成后进入实时监听。

## 二、 核心功能详解

### 1. 自动保存同步 (Update)
- **触发机制**：当您按下 `Ctrl + S` 保存任何文件（`.tscn`, `.gd`, `.tres` 等）时，插件会检测到变更并通过 HTTP 文件通道推送到服务器。
- **接收表现**：
  - 如果您**没有**打开该文件：插件会自动静默更新本地文件。
  - 如果您**正打开**该场景/脚本：插件会弹出冲突询问窗口。

### 2. 远程冲突处理（核心逻辑）
当他人修改了您正在编辑的场景时，会弹出对话框：
- **【确定】(Accept)**：承认他人的修改。您的本地场景会被远程版本覆盖，且场景会自动重载。
- **【取消】(Cancel/Reject)**：**拒绝**他人的修改。
  - **后果**：插件会重新上传您的本地版本，让项目回到您保留的版本。

### 3. 删除同步 (Delete)
- **触发机制**：当您在 Godot 文件系统中删除一个文件时。
- **表现**：所有协作成员的本地对应文件会被同步删除。
- **注意**：如果被删除的文件正好是某人正在打开的场景，该成员会收到弹窗提醒，且该场景会自动关闭。

## 三、 重要注意事项（必读）

### 1. 文件过滤规则
插件**不会**同步以下文件（已内置忽略列表）：
- `.godot/` 文件夹（本地缓存）
- `.git/` 文件夹
- `.import/` 文件夹及 `.import` 后缀文件（每个人的导入设置可能不同）
- `.uid` 文件
- 超过管理员后台设置阈值的文件。默认阈值为 **100MB**。

阈值内的文件都会进入同步。文件内容通过 HTTP 传输，WebSocket 只负责事件通知和在线状态，因此较大的文件不会阻塞实时事件。

### 2. 关于“保留本地”的协作习惯
- 如果冲突弹窗中点击“取消”，插件会保留您的本地版本并重新上传。多人同时修改同一场景时，建议先沟通清楚再选择保留本地。

### 3. 场景刷新延迟
- 文件监听器的扫描间隔为 **1.5秒**。这意味着在您保存后，队友可能需要等待 1-2 秒才会触发同步。

### 4. 首次连接同步
- 当您刚进入项目时，插件会先拉取服务端 manifest 并与本地文件比对：服务端较新的文件会下载，本地较新的文件会上传，双方缺失的文件会补齐。

## 四、 常见问题排查 (FAQ)

- **Q: 为什么我修改了文件队友没收到？**
  - A: 检查右侧面板是否显示“已连接”。另外，请确保您执行了“保存”操作，仅在编辑器里修改而不保存是不会触发同步的。
- **Q: 为什么场景刷新后丢失了节点？**
  - A: 可能是因为引用的资源文件（如贴图、子场景）尚未同步完成。请确保所有关联资源都已保存并成功同步。
- **Q: 服务器断开了怎么办？**
  - A: 插件会自动尝试重连（每3秒一次）。一旦连上，它会自动进行一次全量比对，同步错过的更新。

## 五、 团队协作建议

1. **明确分工**：尽量避免两个人同时修改单个场景。
2. **配合 Git**：DiffFlow 负责**实时**开发，Git 负责**阶段性**归档。
